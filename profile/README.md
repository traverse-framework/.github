# traverse-framework

Traverse is a runtime and composition model for business capabilities — capability contracts, a governed registry, and a local-first execution runtime, built to be portable across execution environments.

## Repos

- [`traverse`](https://github.com/traverse-framework/traverse) — the core runtime, CLI, contracts, and MCP surface
- [`registry`](https://github.com/traverse-framework/registry) — the public capability registry: capability contracts, artifacts, and the publishing pipeline
- [`reference-apps`](https://github.com/traverse-framework/reference-apps) — reference and example applications proving Traverse across web, cloud, mobile, and desktop targets
- [`website`](https://github.com/traverse-framework/website) — the public site and docs
- [`.github`](https://github.com/traverse-framework/.github) (this repo) — shared governance: constitution, NFRs, quality standards, CI gates, CLA, and AI-agent hardening rules used across every repo above

## Governance

Every repo in this org follows the same spec-driven governance model. See [`constitution.md`](../constitution.md) for the full rules, and each repo's own `CLAUDE.md`/`AGENTS.md` for what's specific to that repo.
